# useCallback

<br />

## 1. 왜 등장했을까?

React는 컴포넌트가 리렌더링될 때마다 컴포넌트 본문을 다시 실행한다.

따라서 컴포넌트 내부에서 선언한 함수도 매번 새롭게 생성된다.

일반적으로 함수를 새로 생성하는 비용은 크지 않지만, 함수의 참조가 변경되면 `memo`로 최적화한 컴포넌트가 다시 렌더링되거나 `useEffect`의 의존성이 변경되어 불필요하게 다시 실행될 수 있다.

`useCallback`은 이러한 문제를 해결하기 위해 등장했다.

의존성이 변경되지 않았다면 이전 함수의 참조를 재사용하여 불필요한 리렌더링과 Effect 재실행을 방지할 수 있다.

<br />

## 2. 어떤 훅일까?

```tsx
const handleClick = useCallback(() => {
  console.log("clicked");
}, []);
```

`useCallback`은 함수의 참조를 메모이제이션하는 Hook이다.

의존성이 변경되지 않으면 이전 함수를 그대로 재사용하고, 변경되면 새로운 함수를 생성하여 반환한다.

### 매개변수

- **callback** : 메모이제이션할 함수
- **dependencies** : 함수를 다시 생성할지 판단하는 의존성 배열

### 반환값

- **메모이제이션된 함수**

<br />

## 3. 어떤 상황에서 사용할까?

`useCallback`은 **함수의 참조를 유지해야 하는 경우** 사용한다.

- `memo`로 최적화한 컴포넌트에 함수를 Props로 전달하는 경우
- `useEffect`의 의존성 배열에 함수를 사용하는 경우
- 커스텀 Hook에서 외부로 함수를 반환하는 경우

```tsx
const handleSubmit = useCallback(() => {
  save();
}, [save]);
```

커스텀 Hook에서는 반환하는 함수를 `useCallback`으로 감싸는 것이 권장된다.

이를 통해 Hook를 사용하는 컴포넌트가 `memo`나 `useEffect`를 사용할 때 안정적인 함수 참조를 사용할 수 있다.

<br />

## 4. 사용할 때 주의할 점

`useCallback`은 **성능 최적화를 위한 Hook**이다.

- `useCallback`을 제거해도 프로그램은 정상적으로 동작해야 한다.
- 모든 함수를 `useCallback`으로 감싸는 것은 오히려 성능을 떨어뜨릴 수 있다.
- 의존성 배열에는 함수 내부에서 사용하는 모든 반응형 값을 포함해야 한다.
- 함수 생성 비용을 줄이기 위한 것이 아니라, 함수의 참조를 안정적으로 유지하기 위한 Hook이다.

<br />

## 한 줄 요약

> **useCallback은 함수의 참조를 의존성이 변경될 때까지 유지하여 불필요한 리렌더링과 Effect 재실행을 줄이는 Hook이다.**
