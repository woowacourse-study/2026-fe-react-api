# useMemo

<br />

## 1. 왜 등장했을까?

React는 컴포넌트가 리렌더링될 때마다 컴포넌트 본문을 다시 실행한다.

따라서 렌더링 과정에서 수행하는 계산도 매번 다시 실행된다.

일반적인 계산은 문제가 되지 않지만, 정렬이나 필터링처럼 비용이 큰 계산은 값이 변경되지 않았더라도 반복 수행되어 성능에 영향을 줄 수 있다.

`useMemo`는 이러한 문제를 해결하기 위해 등장했다.

의존성이 변경되지 않았다면 이전 계산 결과를 재사용하여 불필요한 계산을 방지할 수 있다.

<br />

## 2. 어떤 훅일까?

```tsx
const value = useMemo(() => calculate(a, b), [a, b]);
```

`useMemo`는 렌더링 과정에서 계산한 값을 메모이제이션하는 Hook이다.

의존성이 변경되지 않으면 이전 계산 결과를 재사용하고, 변경되면 다시 계산하여 새로운 값을 반환한다.

### 매개변수

- **calculateValue** : 메모이제이션할 계산 함수 (인자를 받지 않는 순수한 함수)
- **dependencies** : 계산 결과를 다시 생성할지 판단하는 의존성 배열

### 반환값

- **메모이제이션된 계산 결과**

<br />

## 3. 어떤 상황에서 사용할까?

`useMemo`는 **비용이 큰 계산 결과를 재사용해야 하는 경우** 사용한다.

- 정렬, 필터링처럼 비용이 큰 연산
- 복잡한 계산을 여러 번 수행하는 경우
- 계산 결과를 자식 컴포넌트에 전달하는 경우

```tsx
const visibleTodos = useMemo(() => {
  return filterTodos(todos, tab);
}, [todos, tab]);
```

의존성이 변경되지 않으면 `filterTodos`를 다시 실행하지 않고 이전 계산 결과를 그대로 사용한다.

<br />

## 4. 사용할 때 주의할 점

`useMemo`는 **성능 최적화를 위한 Hook**이다.

- `useMemo`를 제거해도 프로그램은 정상적으로 동작해야 한다.
- 모든 계산을 `useMemo`로 감싸는 것은 오히려 성능을 떨어뜨릴 수 있다.
- 의존성 배열에는 계산에 사용하는 모든 반응형 값을 포함해야 한다.
- React는 필요에 따라 메모이제이션된 값을 버릴 수 있으므로, 동작을 보장하는 용도로 사용해서는 안 된다.

<br />

## 한 줄 요약

> **useMemo는 렌더링 과정에서 계산한 값을 의존성이 변경될 때까지 재사용하여 불필요한 계산을 줄이는 Hook이다.**
